14 /**
15  * @brief Handle auf eine ERiC-Instanz.
16  *
17  * ERiC-Instanzen werden von der Multithreading-API angelegt,
18  * verwendet und wieder freigegeben, siehe ericmtapi.h.
19  * \n Alle API-Funktionen der Multithreading-API nehmen einen Zeiger auf eine
20  * ERiC-Instanz entgegen und verrichten ihre Aufgaben auf dieser ERiC-Instanz.
21  * Die EricInstanz enthält sämtliche veränderlichen Zustände des ERiC.
22  * Dies sind ERiC-Einstellungen, Plugin- und Log-Verzeichnis, Proxyeinstellungen,
23  * Zertifikatshandle, Rückgabepuffer, etc.
24  * \n Es können mehrere ERiC-Instanzen parallel angelegt werden. Jede dieser
25  * ERiC-Instanzen ist unabhängig von allen anderen ERiC-Instanzen. Verfügen
26  * mehrere Threads jeweils über ihre eigene ERiC-Instanz, können sie diese
27  * parallel verwenden. Dazu müssen die Threads den API-Funktionen der
28  * Multithreading-API ihre jeweils eigene ERiC-Instanz übergeben.
29  * \n ERiC-Instanzen sollen nicht für jede Aufgabe neu erstellt und konfiguriert
30  * werden. Das Erstellen und Zerstören einer ERiC-Instanz ist ressourcen- und zeitintensiv.
31  * Die Lebenszeit einer ERiC-Instanz sollte beispielsweise eher der Lebenszeit
32  * eines Arbeiter-Threads in einem Pool entsprechen, als der Verarbeitungsdauer
33  * einer einzelnen Aufgabe an einen Arbeiter-Thread.
34  * \n ERiC-Instanzen können zwischen Threads ausgetauscht werden. Eine
35  * ERiC-Instanz darf aber nicht in zwei Threads gleichzeitig verwendet werden.
36  *
37  * @see
38  * - ::EricMtInstanzErzeugen()
39  * - ::EricMtInstanzFreigeben()
40  */
41 typedef struct EricInstanz *EricInstanzHandle;

116 /**
117  * @brief Handle zur Verwaltung und Verwendung von Rückgabepuffern.
118  *
119  * Viele ERiC API-Funktionen geben Informationen an ihren Aufrufer zurück,
120  * indem sie Daten in sogenannte Rückgabepuffer schreiben.
121  * Solche Rückgabepuffer müssen mit ::EricRueckgabepufferErzeugen() angelegt
122  * werden. Das bei dieser Erzeugung generierte Puffer-Handle wird vom Aufrufer
123  * an die API-Funktion übergeben, die den Puffer leert bevor sie dann in den
124  * Puffer schreibt. Ein einmal generiertes Puffer-Handle kann damit auch für
125  * mehrere aufeinanderfolgende Aufrufe von ERiC API-Funktionen wiederverwendet
126  * werden.
127  * Mittels ::EricRueckgabepufferLaenge() kann danach die Anzahl der in den
128  * Puffer geschriebenen Bytes ermittelt werden.
129  * Mit ::EricRueckgabepufferInhalt() kann der Pufferinhalt abgefragt werden.
130  * Jeder Rückgabepuffer muss nach seiner Verwendung mit
131  * ::EricRueckgabepufferFreigeben() wieder freigegeben werden.
132  *
133  * Die Struktur EricReturnBufferApi kapselt die Rückgabepuffer-Implementierung.
134  * Anwender der ERiC API verwenden ausschließlich Zeiger auf Instanzen dieser
135  * Struktur und müssen daher deren Felder nicht kennen.
136  *
137  * Rückgabepuffer sind der Singlethreading-API bzw. einer ERiC-Instanz der
138  * Multithreading-API fest zugeordnet. Die Funktionen der ERiC API, die einen
139  * Rückgabepuffer entgegen nehmen, geben den Fehlercode
140  * ::ERIC_GLOBAL_PUFFER_UNGLEICHER_INSTANZ zurück, wenn der übergebene
141  * Rückgabepuffer
142  * - mit der Singlethreading-API erzeugt worden ist und dann mit der
143  * Multithreading-API verwendet wird
144  * - mit der Multithreading-API erzeugt worden ist und dann mit der
145  * Singlethreading-API verwendet wird
146  * - mit einer ERiC-Instanz erzeugt worden ist und dann mit einer
147  * anderen Instanz verwendet wird.
148  *
149  * @see
150  * - ERiC-Entwicklerhandbuch.pdf Kap. "Rückgabepuffer der ERiC Programmierschnittstelle"
151  * - ::EricRueckgabepufferErzeugen()
152  * - ::EricRueckgabepufferLaenge()
153  * - ::EricRueckgabepufferInhalt()
154  * - ::EricRueckgabepufferFreigeben()
155  */
156 typedef struct EricReturnBufferApi* EricRueckgabepufferHandle;
